Recommended folder layout

            
          

          
Recordings made by this app land in a flat <source>/<file> folder automatically - no action needed there. When you're dropping in sets you already have (via the File Explorer, a URL fetch, or just copying files onto the disk), organize them like this so Smart Match can sort them into the right event and stage on its own:

          
recordings/<Event>/<Edition or year>/<Day>/<Stage>/<Event> <year>, <Stage> (<Day>, <date>).mp4

e.g.
recordings/MyFestival/2026/Saturday/MainStage/MyFestival 2026, MainStage (Saturday, 2026-07-04).mp4

          
    
            
  • Only the last two levels are required - the stage folder (the file's immediate parent) and at least one folder above it. The edition/year and day folders are optional; include whichever you have.
    • 
            
  • A YYYY-MM-DD (or similar) date and/or a weekday name in the filename or a day folder lets recordings sort into the right day even without a specific timetable set.
    • 
            
  • This is for a whole day or whole stage recorded as one file, not a single DJ's set - Smart Match won't guess an artist for it. If you've imported an archived timetable, Smart Match still prefers a real per-set match over a folder-based guess.
    • 
            
  • Run Smart Match (above) after adding files: it reads this layout, offers to file each recording under the matching event and stage, and creates the event first if it doesn't exist yet - nothing happens until you approve it.
    •

Peer Sharing (P2P)

              off
            

            
Let another MutiRec instance pull recordings straight from this one. This instance must be reachable at a public URL — set it below and verify it before sharing (the check confirms the URL routes back here). Once verified, use Recordings → Share Sets to generate a code.

            
            
            
Use this if the reachability check fails but you've already confirmed the URL works from outside - this can happen when this instance's own network (e.g. a VPN-gated firewall) reaches itself just fine while real outside clients hit a different path and can't. The check simply can't tell the two cases apart.

            

              
              
              
            

            
Expose this instance publicly however you like — a reverse proxy, a Cloudflare/ngrok tunnel, or a port forward. Share tokens are unguessable and each only exposes the exact files you put in that share; revoke a share any time from Recordings → Share Sets.

Install Notes

          
Run with Docker Compose, mount /data for recordings, and optionally mount network shares into /data/recordings or configure rclone for Dropbox, Google Drive, YouTube archive workflows, or S3-compatible storage.

          
docker compose up -d
docker compose logs -f recorder

          
SMB and NFS are best mounted by the host OS or Docker volume driver, then passed into the container. Raw HTTP sources use ffmpeg directly. Twitch, YouTube, and similar services use streamlink.

          
Hardware transcoding requires the matching device passthrough, for example NVIDIA Container Toolkit for h264_nvenc, Intel Quick Sync for qsv, or VAAPI devices mounted into the container.

          
Live Rewind

          
Turn on "Live rewind" for a source in the Source Manager to let viewers scrub backward while it's actively recording. This runs a second H.264/AAC transcode into a rolling HLS buffer (window length set in Settings), so only enable it where the extra CPU/disk cost is acceptable. Once a recording finishes, the buffer is discarded and the archived file takes over.

          
Timetable

          
The Timetable tab has a visual editor: pick a day, click a set to edit or delete it, or click "+ Set" on a stage row to add one. Raw JSON editing is still available behind "Show raw JSON" for bulk edits. Optionally, look up a festival on timetable.lol and import its schedule instead of building one by hand - this replaces the current timetable and remembers the link so you can re-sync later, but it's entirely optional and never required.

          
Each recording source can optionally be pinned to a specific timetable stage name (Source Manager → "Timetable stage") if it doesn't match the source's own name.

          
Star any set in the timetable to get a reminder (via Discord/SMTP, configured in Settings) a configurable number of minutes before it starts.
