Install Notes

Run with Docker Compose, mount /data for recordings, and optionally mount network shares into /data/recordings or configure rclone for Dropbox, Google Drive, YouTube archive workflows, or S3-compatible storage.

docker compose up -d
docker compose logs -f recorder

SMB and NFS are best mounted by the host OS or Docker volume driver, then passed into the container. Raw HTTP sources use ffmpeg directly. Twitch, YouTube, and similar services use streamlink.

Hardware transcoding requires the matching device passthrough, for example NVIDIA Container Toolkit for h264_nvenc, Intel Quick Sync for qsv, or VAAPI devices mounted into the container.
